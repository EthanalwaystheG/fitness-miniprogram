# 功能需求文档

## 项目概述
健身记录微信小程序，帮助用户记录每日训练和饮食，追踪体重变化，了解自身基础代谢水平。

## 功能模块

### 1. 今日概览 (Dashboard)
- **日期显示**: 当前日期（XXXX年XX月XX日格式）
- **四卡片概览**: 每日消耗(TDEE)、剩余热量、基础代谢(BMR)、最新体重
- **快捷操作**: 一键跳转记录训练、记录饮食、记录体重
- **训练摘要**: 今日训练动作数、总组数
- **饮食摘要**: 今日摄入总热量、宏量营养素占比条
- **空状态**: 首次使用引导设置个人信息
- **更新机制**: 每次切换到该 Tab 自动刷新数据

### 2. 训练记录 (Workout)
- **日期选择**: 日期 picker 选择任意日期查看/编辑
- **动作列表**: 展示当天所有训练动作（名称、组数×次数、重量、备注）
- **新增动作**: 弹窗表单（动作名称、组数、次数、重量、备注）
- **编辑动作**: 点击编辑修改已有动作
- **删除动作**: 确认后删除，最后一个动作删除则清除当日记录
- **统计栏**: 动作总数、总组数
- **空状态**: 无记录时引导添加

### 3. 饮食记录 (Diet)
- **日期选择**: 日期 picker 选择任意日期查看/编辑
- **四餐分段**: 早餐、午餐、晚餐、加餐
- **食物列表**: 每餐下展示食物（名称、份量、热量、宏量素）
- **新增食物**: 弹窗表单（名称、份量、单位、热量、蛋白质、碳水、脂肪）
- **编辑/删除食物**: 支持修改和删除
- **macro-bar**: 每餐和全天的蛋白质/碳水/脂肪占比可视化
- **每日总计**: 总热量 + 宏量素汇总
- **空状态**: 无记录时引导添加

### 4. 个人资料 & 体重 (Profile)
- **个人信息表单**: 身高(cm)、体重(kg)、年龄、性别(男/女)、活动水平(5档)
- **BMR/TDEE 实时计算**: 表单数据变化即时更新
- **体重录入**: 输入当日体重保存
- **体重趋势图**: 折线图展示历史体重变化（纯 view 实现）
- **体重历史列表**: 按日期倒序展示，显示每笔体重与上一笔的差值
- **删除体重**: 支持删除单条记录

## 数据模型

### 用户信息 (UserProfile)
| 字段 | 类型 | 说明 |
|------|------|------|
| height | number | 身高 cm (100-250) |
| weight | number | 体重 kg (30-300) |
| age | number | 年龄 (10-120) |
| gender | 'male' \| 'female' | 性别 |
| activityLevel | 1.2/1.375/1.55/1.725/1.9 | 活动系数 |
| updatedAt | string | ISO 更新时间 |

### 体重记录 (WeightRecord)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| date | string | YYYY-MM-DD |
| weight | number | 体重 kg |
| createdAt | string | ISO 创建时间 |

### 训练记录 (WorkoutRecord)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| date | string | YYYY-MM-DD |
| exercises | Exercise[] | 动作列表 |
| createdAt/updatedAt | string | ISO 时间 |

**Exercise**: { id, name, sets, reps, weight, note }

### 饮食记录 (DietRecord)
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| date | string | YYYY-MM-DD |
| meals | Meal[] | 四餐列表 |
| createdAt/updatedAt | string | ISO 时间 |

**Meal**: { id, type('breakfast'\|'lunch'\|'dinner'\|'snack'), foods: FoodItem[] }
**FoodItem**: { id, name, calories, protein, carbs, fat, amount, unit }

## 非功能需求
- 纯本地存储，无需网络连接
- Skyline 渲染引擎兼容
- 基础库 2.32.3 及以上
- 数据隔离：不同日期的训练/饮食记录互不影响
